# Global Context

## Documentation

- If code is self-explanatory, avoid adding superfluous comments. Let code self-document itself.
- If code is not self-explanatory, add concise comments to clarify its purpose and functionality.
- Add comments prefixed with "XXX: " for pitfalls or important considerations.
- Always review pre-existing code before adding new code to ensure consistency.
- Never stage or commit any changes to Git unless explicitly asked to do so.
